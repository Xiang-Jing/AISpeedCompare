# LangGraph

## 项目介绍

LangGraph 是 LangChain 生态下的开源 Agent 编排框架，用于构建可持久化、可恢复、可观测的状态化 Agent 工作流。它不是 GUI 自动化或 RPA 框架，但适合承载“AI 探索 -> 轨迹分析 -> 参数抽取 -> 脚本生成 -> 回放验证 -> 人审发布”这一类长流程。

对当前问题来说，LangGraph 的价值在于：不要让一个大模型提示词一次性完成全部事情，而是把生成参数化脚本拆成多个有状态节点，并让每个节点有明确输入、输出、重试和人工确认。

## 录制方式

- LangGraph 本身不录制桌面操作。
- 它可以编排其他工具完成录制或探索，例如调用 Midscene、UFO、UI-TARS、pywinauto/FlaUI、Playwright、OCR 和脚本校验器。
- 可以把每次 Agent 探索轨迹保存为状态，并在失败后从中间节点恢复。
- 可以在关键节点加入人工审核，例如确认参数定义、确认脚本影响范围、批准发布到脚本库。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 依赖接入的 LLM/VLM |
| 框架自动截图 | 不原生提供，可编排桌面/浏览器工具 |
| 人工录制 | 不提供 |
| 生成回放资产 | 可编排 LLM 生成脚本/DSL |
| 参数化和批量执行 | 适合作为参数抽取、验证和发布流程编排层 |
| 一键回放 | 不直接执行 GUI，可调度执行器 |
| 自动探索 App | 依赖下游 GUI Agent |
| 人审节点 | 支持以工作流方式集成 |
| 开源 | 是，MIT |

## 优势

- 适合长流程 Agent，不必把探索、生成、验证都塞进一次模型调用。
- 有状态、可恢复，适合失败重试和审计。
- 可以把“生成脚本”拆成独立节点：任务解析、环境探测、元素识别、参数抽取、IR 生成、脚本转译、沙箱回放、差异分析、人审发布。
- 可与 Python 生态、数据库、对象存储、测试框架和调度系统集成。

## 局限和风险

- 不是 RPA 产品，也不能直接操作桌面软件。
- 需要开发工作流节点和工具适配器。
- 如果没有强约束的 IR、校验器和沙箱，Agent 仍可能生成不稳定或不可审计的脚本。

## 不符合要求的核心原因

1. **它是编排层，不是录制回放执行层。**
2. **不能替代 pywinauto/FlaUI/OpenRPA/Playwright 这类确定性执行工具。**
3. 需要与 GUI Agent 和脚本生成器组合使用。

## 适用结论

LangGraph 建议作为方案里的“Agent 生产流水线编排层”。它负责组织 AI 探索、轨迹归纳、脚本生成、自动回放验证和人工审批，使最终产物从“模型临时操作”变成“可版本管理的参数化脚本”。
