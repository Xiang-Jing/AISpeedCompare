# UI-TARS Desktop

## 项目介绍

UI-TARS Desktop 是开源的多模态 GUI Agent。用户用自然语言描述任务后，Agent 自动截取屏幕，由视觉模型识别界面元素、规划操作步骤，并控制鼠标和键盘完成任务。

它更像“会看屏幕并自己操作的智能体”，而不是传统录制器。

## 录制方式

- **不要求先录制。** 用户直接用文字描述任务即可。
- 执行过程中，框架自动连续截取当前屏幕，视觉模型根据截图决定下一步操作。
- 这些截图属于 Agent 的实时观察，不是让用户手工截取按钮模板。
- 它也不是先录一段视频，再由模型把视频转换为回放脚本。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 支持 |
| 框架自动截图 | 支持 |
| 人工录制 | 不需要；以文字任务和运行时自动截图为主 |
| 生成确定性脚本 | 暂无成熟闭环 |
| 一键回放 | 可重新提交任务执行，但不等同固定脚本回放 |
| 自动探索 App | 支持，可观察界面并自主规划 |
| 适用平台 | Windows、macOS、浏览器 |

## 优势

- 不需要为每个按钮人工截取图片模板。
- 能理解自然语言目标和屏幕内容。
- 面对陌生界面时具备一定探索和路径规划能力。
- 适合验证“给出数据目标后自行寻找入口”的未来形态。

## 局限和风险

- 同一任务多次执行可能产生不同路径。
- 长流程、动态菜单、弹窗和滚动列表可能出现偶发误操作。
- 目前更侧重 Agent 直接执行，不侧重生成可审阅、可版本管理的脚本。
- 模型调用会增加时延和成本。

## 不符合要求的核心原因

1. **没有成熟的“执行或录制一次后，生成独立脚本并稳定回放”闭环。**
2. 重新执行自然语言任务属于再次推理，不是确定性回放。
3. 对数据采集生产任务而言，可审计性和成功率仍需专门验证。

## 适用结论

适合自动探索 App 和 AI 操作 PoC。若用于生产，建议把探索结果再转换为 Playwright、Python 或其他确定性脚本。
