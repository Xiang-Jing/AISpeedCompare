# OpenRPA

## 项目介绍

OpenRPA 是开源的传统 RPA 平台，定位接近开源版 UiPath。它提供可视化流程设计、录制、UI 元素选择和工作流执行能力，可以将人工操作保存为 XAML 工作流后重复运行。

## 录制方式

- **主要方式：操作事件录制。** 记录点击、输入和选中的 UI 元素，并转换为 XAML 工作流活动。
- **不是视频录制。** 回放依赖工作流、选择器和动作参数，不是分析录像。
- 标准控件通常不需要截图；只有采用图像匹配时才需要模板截图。
- **不能只给一段文字描述。** 它不能自主理解业务目标并探索 App。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 不原生支持 |
| 框架自动截图 | 支持 |
| 人工录制 | 支持操作事件录制并生成 XAML；不是视频学习 |
| 生成回放资产 | 支持，XAML 工作流 |
| 一键回放 | 支持 |
| 自动探索 App | 不支持 |
| 适用平台 | 重点是 Windows，也支持 Web、SAP 等集成 |

## 优势

- 录制和回放链路完整。
- 工作流可视化，适合非纯代码团队维护。
- 支持 UI 元素、窗口、浏览器和企业软件自动化。
- 可以保存、修改、调度和重复执行流程。

## 局限和风险

- 核心是规则驱动，不会理解“帮我找到某类数据”这类开放目标。
- 面对未知页面时不能自行探索路径。
- 特殊桌面控件仍可能需要 OCR、图像或坐标兜底。
- 社区活跃度低于当前热门 AI Agent。

## 不符合要求的核心原因

1. **不具备原生 AI 语义理解。**
2. **不具备根据数据目标自主探索 App 的能力。**
3. 只能高质量执行已经录制或配置好的流程。

## 适用结论

适合作为确定性录制回放的基线方案，可与外部 AI Agent 组合，但单独使用不满足目标。
