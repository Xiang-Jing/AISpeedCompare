# OpenAdapt

## 项目介绍

OpenAdapt 是面向 GUI 自动化的开源 AI 项目。它自动记录用户的鼠标、键盘、活动窗口和屏幕状态，再利用多模态模型理解示教过程，并尝试在界面变化后自适应回放。

## 录制方式

- **主要方式：人工示教录制。**
- 录制时自动采集鼠标事件、键盘事件、活动窗口信息以及连续截图/屏幕状态。
- 从效果上看像录制用户操作，但底层保存的是结构化事件和屏幕图像，不是单纯保存一段普通视频。
- 用户不需要逐个截取按钮模板；框架自动捕获屏幕上下文。
- 文字目标可辅助理解和回放，但其核心仍是先有人演示一遍。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 支持 |
| 框架自动截图 | 支持 |
| 人工示教录制 | 支持事件和连续截图采集；不是普通视频录像 |
| 生成回放资产 | 支持保存 Demonstration/Recording |
| 一键回放 | 支持 AI 回放策略 |
| 自动探索 App | 部分支持，不是核心强项 |
| 适用平台 | Windows、macOS 和 Web GUI |

## 优势

- 不需要像图片模板工具一样人工逐个截取按钮。
- 最接近“人操作一次，AI 学会后再执行”。
- 回放时可以根据当前界面重新识别元素，不完全依赖固定坐标。
- 适合桌面 AI 示教回放 PoC。

## 局限和风险

- 首次流程通常仍需要人工完整示教。
- 强项是模仿示教，不是根据开放数据目标自行探索陌生软件。
- 回放资产更接近录制数据和策略，不如 Python/YAML 脚本容易审阅。
- 长流程稳定性、敏感截图存储和模型成本需要实测。

## 不符合要求的核心原因

1. **自动探索能力有限，核心范式仍是人工先示教。**
2. 不一定生成独立、确定性、容易版本管理的业务脚本。
3. 生产成熟度和复杂桌面任务成功率需要验证。

## 适用结论

当前最值得验证的桌面 AI 示教回放方案之一，但若未来重点是“只描述数据目标，系统自己探索”，还需要叠加更强的规划 Agent。
