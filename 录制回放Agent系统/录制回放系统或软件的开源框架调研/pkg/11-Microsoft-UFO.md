# Microsoft UFO/UFO²

## 项目介绍

Microsoft UFO 是面向 Windows 的开源多模态 Agent。它根据自然语言任务分析桌面截图和 Windows 控件信息，规划操作步骤，并控制一个或多个应用完成任务。

UFO² 在这一方向上进一步强化多应用协作、任务分解、控件交互和经验复用。

## 录制方式

- **不要求人工录制。** 用户直接用文字描述任务或希望获得的数据。
- 执行中框架自动截取屏幕，并结合 UIA、Win32/COM 等控件信息观察应用。
- 不需要用户手工制作截图模板。
- 不是先录制视频再生成脚本，也不是传统鼠标键盘录制器。
- 可以复用任务经验，但经验复用不等于导出确定性回放脚本。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 支持 |
| 框架自动截图 | 支持 |
| UIA/原生控件信息 | 支持 |
| 人工录制 | 不需要；以文字任务、自动截图和控件信息为主 |
| 生成确定性脚本 | 暂无成熟通用闭环 |
| 一键回放 | 可重新运行任务和复用经验，但不等同固定脚本 |
| 自动探索 App | 支持 |
| 开源 | 是，MIT |

## 优势

- 专门面向 Windows 桌面应用。
- 可以同时利用视觉、UIA、Win32/COM 等信息，比纯坐标点击更可靠。
- 能理解开放式任务并探索多个应用。
- 与“给出数据目标，自己寻找应用入口”的未来方向一致。

## 局限和风险

- 核心是 Agent 自主执行，不是传统录制器。
- 每次执行可能重新规划，确定性和可复现性弱于固定脚本。
- 部署、模型、权限和跨应用任务配置较复杂。
- 对专业金融客户端的控件兼容性需要实测。

## 不符合要求的核心原因

1. **缺少成熟的录制后自动生成独立脚本并稳定回放的闭环。**
2. 任务和经验复用不等同于可审阅、可版本管理的确定性脚本。
3. 自主探索能力较强，但生产成功率和错误恢复仍需验证。

## 适用结论

适合验证 Windows App 自主探索能力。推荐与 pywinauto、Robot Framework 或 Python 脚本生成层结合，把探索结果固化后再生产回放。
