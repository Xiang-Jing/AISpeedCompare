# CUA-Skill

## 项目介绍

CUA-Skill 是微软研究提出的 Computer Use Agent 技能化框架，核心目标不是让 Agent 每次都重新看屏幕、重新推理，而是把成功完成过的 GUI 任务沉淀为可复用的技能。它把任务表示为参数化执行图（Parameterized Execution Graph，PEG），并把不同界面状态、动作、参数、条件和恢复路径组织成技能资产。

这条路线非常贴近当前痛点：AI 可以负责探索和生成初版技能，但生产阶段应尽量执行已经固化、可校验、可参数化的技能图，而不是每次依赖模型临场发挥。

## 录制方式

- **主要方式：从任务轨迹、动作序列和界面状态中构建技能。**
- 技能不是简单鼠标键盘录制，而是把操作过程抽象成状态、动作、参数、条件和恢复逻辑。
- 论文和项目页面强调可复用技能、参数化和执行图，适合作为“Agent 录制结果如何沉淀”的架构参考。
- 当前更偏研究框架，不是开箱即用的 RPA Studio。
- 如果要落地到竞品数据采集，需要自研“轨迹采集 -> 技能图生成 -> 脚本导出 -> 回放验证”的工程链路。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 支持，依赖外部 CUA/LLM/VLM |
| 框架自动截图 | 作为 computer-use agent 轨迹输入的一部分 |
| UIA/原生控件信息 | 可作为状态表示来源，具体实现需自建 |
| 人工录制 | 不是传统录制器；更强调从轨迹抽象技能 |
| 生成确定性脚本 | 不是现成脚本生成器，但 PEG 思路适合转译为 DSL/Robot/Python |
| 参数化和批量执行 | 是其核心思想之一 |
| 一键回放 | 技能可复用，但工程成熟度需自建验证 |
| 自动探索 App | 依赖上层 Agent |
| 开源 | 研究代码/项目页面公开，需核实具体许可证与可商用边界 |

## 优势

- 非常契合“AI 生成参数化脚本”这个问题的中间表示设计。
- 能把一次成功探索抽象成可复用技能，减少每次运行都让 Agent 自由探索的风险。
- 参数、条件分支和恢复路径天然适合表达“同一竞品数据流程，换证券代码、日期、指标、导出路径重复执行”。
- 可作为自研 DSL 的蓝本：先存技能图，再导出 Python、Robot Framework、Playwright、OpenRPA XAML 或 AHK。

## 局限和风险

- 不是成熟 RPA 产品，也不是可直接部署的企业录制回放平台。
- 需要自建轨迹采集、技能图编辑、参数抽取、验证、导出和审计。
- 对具体桌面软件的控件定位仍要依赖 pywinauto/FlaUI/OCR/视觉等执行层。
- 研究代码和论文思路需要工程化落地，不能直接当作生产主链路。

## 不符合要求的核心原因

1. **它提供的是技能化思想和研究实现，不是完整产品。**
2. **没有现成的“点一下生成生产级参数化脚本”的闭环。**
3. 需要结合 Agent 探索层和确定性执行层，才能形成可用系统。

## 适用结论

CUA-Skill 不适合作为单独工具使用，但非常适合作为本项目的核心方法论：把 Agent 探索轨迹沉淀为“参数化技能图”，再由编译器生成稳定回放脚本。

建议在最终方案中把它放在“技能中间层 / IR 层”：上接 Midscene、UFO、UI-TARS、Agent S 等探索 Agent，下接 Python、Robot Framework、Playwright、FlaUI、OpenRPA 等确定性执行器。
