# Python + pywinauto + pyautogui + OCR

## 项目介绍

这是由多个开源组件组成的自研桌面自动化方案，不是一个开箱即用的产品：

- `pywinauto`：通过 Win32/UIA 操作标准 Windows 控件。
- `pyautogui`：控制鼠标、键盘并自动截图。
- PaddleOCR 等 OCR：识别界面文字。
- OpenCV：图像模板匹配。
- LLM/VLM：理解自然语言目标、截图并生成脚本。

## 录制方式

- 该组合默认没有录制器，通常由开发者编写 Python，或让 LLM 根据文字需求生成初版代码。
- `pyautogui` 等组件可以自动截图，但截图只是运行时输入或图像定位材料。
- 可以自行开发“鼠标键盘事件 + 连续截图”的示教录制器。
- 也可以自行录制视频，但现有组件不会自动理解视频并生成可靠回放脚本；若需要该能力，必须额外建设视频解析、动作识别和脚本生成链路。

## 推荐定位顺序

| 定位方式 | 稳定性 | 建议 |
| --- | ---: | --- |
| UIA/Win32 控件定位 | 高 | 首选 |
| OCR 文字定位 | 中高 | 菜单、候选项和中文按钮 |
| 图像模板匹配 | 中 | 固定图标和无文字按钮 |
| 相对坐标 | 中低 | 兜底 |
| 绝对坐标 | 低 | 尽量避免 |

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 可以实现，需要接入 LLM/VLM |
| 框架自动截图 | 支持 |
| 人工录制 | 默认没有；事件、截图序列或视频录制均需自行开发 |
| 生成回放资产 | 支持 Python，也可自建 YAML/DSL |
| 一键回放 | 支持 |
| 自动探索 App | 可以实现，需要自建 Agent |
| 开源 | 关键组件可全部开源 |

## 可建设的目标架构

```text
自然语言数据目标
       ↓
LLM/VLM 观察截图并规划
       ↓
优先调用 UIA，其次 OCR/视觉
       ↓
探索出可行操作路径
       ↓
生成 YAML/DSL 或 Python 脚本
       ↓
人工审核、保存、重复回放
```

## 优势

- 技术路线完全可控，可部署在内网。
- 能把 AI 探索结果固化成确定性 Python 脚本。
- 可针对 Wind、同花顺等特定软件做深度适配。
- 易于接入数据库比对、日志和监控。

## 局限和风险

- 录制器、Agent、DSL、权限控制、异常恢复都需要自行建设。
- GUI 自动化需要大量真实场景调试。
- AI 生成代码必须审核，避免误操作和数据泄露。

## 不符合要求的核心原因

1. **它不是现成的框架或软件包，而是一套自研技术组合。**
2. 默认不具备 AI 语义理解和自动探索，需要额外开发。
3. 首次建设成本、测试成本和长期维护成本最高。

## 适用结论

如果长期目标是“AI 先探索、再生成稳定脚本回放”，该路线最可控，但不适合追求立即开箱使用的场景。
