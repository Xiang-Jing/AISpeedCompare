# Power Automate Desktop

## 项目介绍

Power Automate Desktop 是 Microsoft 提供的 Windows 桌面 RPA 工具。它通过桌面录制器、Web 录制器和可视化流程设计器，把鼠标、键盘、窗口、浏览器、Excel、文件等操作保存为 Flow，再按流程重复执行。

它不是开源项目，但可以作为成熟商业 RPA 的对照方案。

## 录制方式

- **主要方式：操作事件录制。** 用户实际点击、输入、切换窗口时，录制器把动作和 UI 元素转换成 Flow 步骤。
- **不是视频录制生成流程。** 即使开启视频或截图留证，也不是依赖模型理解视频来生成回放。
- 通常不需要人工截图；只有标准 UI 元素无法识别、改用图像匹配时，才需要准备模板截图。
- 加入 Copilot 后可以用文字描述辅助创建流程，但该能力依赖 Microsoft 商业产品。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 部分支持，需要 Copilot、AI Builder 等 Microsoft 商业能力 |
| 框架自动截图 | 支持 |
| 人工录制 | 支持操作事件录制；不是视频学习 |
| 生成回放资产 | 支持，生成可视化 Flow |
| 一键回放 | 支持 |
| 自动探索 App | 较弱，主要按已经设计好的 Flow 执行 |
| 开源 | 否 |

## 优势

- Windows、Office、Excel 和浏览器集成成熟。
- 可以录制操作并生成可视化流程。
- 支持异常处理、条件、循环、OCR、图像识别和 UI 元素定位。
- 非开发人员也能维护流程。

## 局限和风险

- 不是开源方案，授权、企业治理和云端 AI 成本需要评估。
- Copilot 生成流程不等于 Agent 自主探索应用。
- 对 Canvas、远程桌面、特殊绘制控件和动态菜单仍可能需要图像或坐标定位。

## 不符合要求的核心原因

1. **不符合开源要求。**
2. AI 语义理解依赖 Microsoft 商业能力，不是可独立部署的开源模块。
3. 主要能力是录制和回放既定流程，不擅长根据“我要什么数据”自行探索陌生 App。

## 适用结论

适合作为传统商业 RPA 对照组，不建议作为本次开源 AI 自动化方案。
