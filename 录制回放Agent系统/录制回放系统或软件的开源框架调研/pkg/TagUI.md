# TagUI

## 项目介绍

TagUI 是 AI Singapore 推出的开源 RPA 工具，使用接近自然语言的文本 DSL 描述浏览器和桌面操作。这里的“自然语言风格”是固定语法，并不代表它能够通过 AI 理解任意业务语义。

## 录制方式

- **主要方式：编写 TagUI 文本 DSL。**
- 某些辅助工具可以帮助生成步骤，但不是 AI 理解任意文字目标。
- 视觉自动化步骤可能需要用快照工具制作按钮或区域的模板截图。
- 不支持录制视频后自动理解并生成回放脚本。

## 核心能力

| 能力 | 结论 |
| --- | --- |
| AI 语义理解 | 不支持 |
| 框架自动截图 | 支持 |
| 人工录制 | 主要编写文本 DSL；视觉步骤可能使用模板截图 |
| 生成回放资产 | 支持，TagUI 文本脚本 |
| 一键回放 | 支持 |
| 自动探索 App | 不支持 |
| 开源 | 是，Apache-2.0 |

## 优势

- 免费开源。
- DSL 比 Python 更容易入门。
- 支持浏览器、OCR、视觉自动化和简单桌面操作。
- 适合快速制作固定流程原型。

## 局限和风险

- 属于传统规则型 RPA。
- 无法理解开放式数据目标并自行寻找入口。
- 复杂 Windows 应用和异常分支的维护能力有限。
- 正式 Release 停留在 2022 年，活跃度不符合“近期持续迭代”的偏好。

## 不符合要求的核心原因

1. **不具备 AI 语义理解。**
2. **不具备自主探索 App 的能力。**
3. 虽然能回放脚本，但只能执行预先写好的固定步骤。
4. 正式版本活跃度较低。

## 适用结论

适合简单固定流程，不建议作为未来 AI 探索型自动化底座。
