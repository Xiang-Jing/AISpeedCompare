# 录制回放系统或软件的开源框架调研清单

## 排序依据

本清单按当前任务目标排序：优先寻找**支持桌面软件、能够 AI 驱动探索、并尽量生成可保存/可回放脚本或工作流**的框架。

排序权重参考：

1. 是否支持桌面软件，尤其是 Windows 桌面应用。
2. 是否具备 AI 语义理解和自动探索能力。
3. 是否能把探索或录制结果保存为可回放资产，最好是 JS、YAML、Python、Robot 或 XAML 等可审阅脚本/工作流。
4. GitHub Stars、社区热度、近期版本/迭代活跃度。
5. 是否开源、是否便于私有化部署和二次开发。

## 建议结论

优先 PoC 建议按以下组合推进：

1. **Midscene**：作为第一候选，验证 AI 探索桌面软件、自动截图理解界面、生成 JS/YAML 回放资产的完整链路。
2. **Microsoft UFO/UFO² 或 UI-TARS Desktop**：作为桌面 AI 探索能力对照，验证它们能否在目标软件中稳定找到入口和路径。
3. **Python + pywinauto + pyautogui + OCR**：作为兜底工程化路线，把 Agent 探索出的路径固化为可审阅、可调试、可版本管理的 Python/DSL 脚本。
4. **Robot Framework/RPA Framework**：作为后续生产化回放、日志、断言和报告层。
